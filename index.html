刷题-剑指

Posted on 2019-10-09 | Edited on 2019-10-10

题目

一个整型数组里除了两个数字之外，其他的数字都出现了两次。请写程序找出这两个只出现一次的数字。

解答

学习到的解法

思路：

  1. 首先array 全部异或后， bit = a ^ b；
  2. bit 二进制中 = 1的位置就是a、b中不同的位置；我们找到一个位置，不一定是第一个，我们取第一个，例如 index = 3；
  3. 将数组根据 二进制index = 3位置 是否 = 1来分为两组；

[分析] 如果数组 {1, 2, 2, 3, 3, 4, 4, 5}, a = 1， b = 5;

  1. 1 与 5 的 index位置 不同，而重复数字2、3、4 index 位置肯定是相同的，无论它index 位置 是 0 还是 1；

  2. 即 a、b 肯定分到不同组，而 两个相同数字肯定分到同一组；

  3. 两组分别组内异或，最终重复的抵消，只剩下a 和 b;

可学习小技巧:

1. (item >> index) & 1 == 1 直接检验item 的 第 index 位置是否是1；

扩展（讨论中看到的）

题目：
数组a中只有一个数出现一次，其他数字都出现了3次，找出这个数字。

一个比较通用的只出现一次的数字的方法

思路 >>：a 代表出现一次的数字

  1. bits[] 统计所有数字，二进制上 每一位 1的个数；
  2. != 3 的代表 a 加上去的次数，代表此index, a 的二进制 为1, 根据这个计算出a;

题目

小明很喜欢数学,有一天他在做数学作业时,要求计算出9~16的和,他马上就写出了正确答案是100。但是他并不满足于此,他在想究竟有多少种连续的正数序列的和为100(至少包括两个数)。没多久,他就得到另一组连续正数和为100的序列:18,19,20,21,22。现在把问题交给你,你能不能也很快的找出所有和为S的连续正数序列? Good Luck!
输出描述:
输出所有和为S的连续正数序列。序列内按照从小至大的顺序，序列间按照开始数字从小到大的顺序

关键字： 连续正数 的 sum; 至少包括两个数；

解答

从别人答案中学到的思路，写出内容

思路：

  1. 窗口机制， left、right 维持一个窗口，sum 符合就加入result; 大于sum 就left++， 小于就right++;

  2. 原本我写的 判断条件 right < sum, 别人答案中是 left < right 。

因为至少包含两个数字，所以left < right;
我自己想法 right 一定要小于sum；

如果 right <= sum 或者 left <= right，就会包含 sum = 3, [[1,2], [3]] 这种情况了；

可学习小技巧:

连续序列求和 (a + b) * (b - a + 1) / 2 = sum

题目

输入一个递增排序的数组和一个数字S，在数组中查找两个数，使得他们的和正好是S，如果有多对数字的和等于S，输出两个数的乘积最小的。

输出描述:
对应每个测试案例，输出两个数，小的先输出。

解答

思路：

  1. HashSet 存储遇到的数字，遍历 array 时候 之前如果遇到过 符合 sum - item 时候，代表符合我们要找的结果；
  2. 将这个预选结果 与 当前 result 中存储的 结果，比较它们的乘积；

Note：result.clear(); 一定要加不然，数组会有多项内容，就算每次都是
result.add(0, sum - array[i]);这种指定位置也不行；

感想：没有发挥出 数组本身有序的优势，HashSet 是额外引入；

讨论：

根据别人思路写出的解答

思路：

  1. 无论有没有负数，a、b位置相差越远，乘积越小；
  2. 数列递增

感想：比我原本思路优秀很多，需掌握；相差越远乘积越小这个概念没想到；

消息通信相关

Posted on 2019-10-09 | Edited on 2019-10-10

基本信息

阅读书籍：《Android 设计与实现 卷1》 第四部分 消息通信篇

所得1：

Java 层的MessageQueue 与 Native 层的 NatveMessageQueue 专门有方法将它们关联，确保Java层可以访问到Native层。

Java层 与 Native层 均有 Looper、MessageQueue 对象，通过Native 方法关联了 双方MessageQueue。

Java 层的MessagQueue 成员变量保存 Native 层的 NatveMessageQueue, 访问时候 通过JNI访问。

所得2：

Handler 使用 成员变量mMessenger 用于跨进程发送（待扩展）

所得3：

Message 本身 存在next属性指向下一条，形成链表；

消息池

Message中 存在消息池，消息池Message sPool 也是一个Message对象，obtain方法就是取出头部消息；

recycle方法 将消息加入消息池头部，并清除这个消息的数据记录；

所得4：

消息在消息队列中 按照执行时间排列，每个消息加入队列就会放到正确位置；

  1. 队列 == null, 放到头部；
  2. when = 0，放头部，需立即处理；
  3. when < 头部.when, 放头部；
  4. 新消息 晚于头部消息，遍历队列插入合适位置；

消息加入队列后，处于block状态，需要等到nativeWake方法唤醒 looper 那边；

唤醒：

第一步：JNI 层Looper对象创建时候，会创建一个 管道写端，写入'W';

第二步：

  • Looper.loop() 中通过 queue.next 获取下一个信息;
  • MessageQueue.next() 中，通过nativePollOnce()监控管道事件做不同处理；

所以 MessageQueue在必要使用通过管道写入 一个值，Looper 这边 监听管道写入 得到唤醒。

所得5 关于空闲状态消息处理

空闲状态：
消息队列为空； 队列头部处理时间未到；

MessageQueue 有一个成员变量，存储IdleHandler， 提供了我们可以add、remove IdleHandler方法，这个handler 用来空闲期处理空闲任务；

next()中会执行nativePollOnce() 根据它返回的状态决定要不要执行空闲任务。如果判断执行，我们添加过 IdleHandler 就执行 IdleHandler.queueHandler();
IdleHandler 是一个接口，需实现queueHandler()

作用：

  • GcIdle中空闲时内存垃圾回收
  • Home启动后，发送BOOT_COMPLETED广播,后面章节分析；
